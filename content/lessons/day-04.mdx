## Function Composition

`pipe` and `flow` are fundamental to the fp-ts style. They let you chain transformations left-to-right without intermediate variables.

## pipe vs flow

```typescript
import { pipe, flow } from 'fp-ts/function'

const double = (n: number) => n * 2
const addOne = (n: number) => n + 1
const toString = (n: number) => `Result: ${n}`

// pipe: eagerly applies value through functions
const result = pipe(5, double, addOne, toString)
console.log(result) // "Result: 11"

// flow: creates a new function (lazy)
const transform = flow(double, addOne, toString)
console.log(transform(5)) // "Result: 11"
```

Use `pipe` when you have a value ready. Use `flow` when defining a reusable transformation function.
