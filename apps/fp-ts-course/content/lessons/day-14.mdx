How do you sort a list of users by last name, then by first name, then by age? How do you check if two complex objects are equal by value rather than by reference? How do you combine a list of optional numbers, skipping the empty ones? These are questions about comparison, equality, and combination — operations that appear in almost every application. This lesson covers the fp-ts type classes that model these operations in a principled, composable way.

## Introduction

Every time fp-ts's `Array.sort`, `Option.getEq`, or `Map` operations work correctly, there is a typeclass instance behind the scenes. Understanding the four core typeclasses — `Eq`, `Ord`, `Semigroup`, and `Monoid` — unlocks the full library.

These are not abstract mathematical curiosities. They are the interfaces that fp-ts uses to make generic operations work across any type. Once you understand them, you can:

- Sort arrays of any type using `Ord`
- Check structural equality (not reference equality) using `Eq`
- Combine values using `Semigroup`
- Fold entire collections to a single value using `Monoid`

## Eq — Structural Equality

`Eq<A>` is an interface with one method: `equals(a: A, b: A): boolean`. It defines what "equal" means for a type.

```typescript
import * as Eq from 'fp-ts/Eq'

// Built-in Eq instances
import { Eq as EqString } from 'fp-ts/string'
import { Eq as EqNumber } from 'fp-ts/number'

EqString.equals('hello', 'hello') // true
EqNumber.equals(1, 2)             // false
```

### Eq.fromEquals — create a custom Eq

```typescript
type User = { id: number; name: string }

// Two users are equal if they have the same ID (regardless of name)
const EqUser: Eq.Eq<User> = Eq.fromEquals(
  (a, b) => a.id === b.id
)

EqUser.equals({ id: 1, name: 'Alice' }, { id: 1, name: 'Alice V2' })  // true
EqUser.equals({ id: 1, name: 'Alice' }, { id: 2, name: 'Alice' })      // false
```

### Eq.contramap — derive Eq from another

```typescript
import { contramap } from 'fp-ts/Eq'
import { Eq as EqNumber } from 'fp-ts/number'
import { pipe } from 'fp-ts/function'

// Derive Eq<User> from Eq<number> by comparing IDs
const EqUserById: Eq.Eq<User> = pipe(
  EqNumber,
  contramap((user: User) => user.id)
)
```

### Where Eq appears in fp-ts

```typescript
import * as O from 'fp-ts/Option'

// Option.getEq needs an Eq for the inner type
const EqOptionUser: Eq.Eq<O.Option<User>> = O.getEq(EqUser)

EqOptionUser.equals(O.some({ id: 1, name: 'Alice' }), O.some({ id: 1, name: 'Bob' })) // true (same id)
EqOptionUser.equals(O.some({ id: 1, name: 'Alice' }), O.none)                         // false
```

## Ord — Ordering and Comparison

`Ord<A>` extends `Eq<A>` with a `compare(a: A, b: A): Ordering` method, where `Ordering` is `-1 | 0 | 1`. It defines what "less than, equal to, greater than" means.

```typescript
import * as Ord from 'fp-ts/Ord'
import { Ord as OrdString } from 'fp-ts/string'
import { Ord as OrdNumber } from 'fp-ts/number'

OrdNumber.compare(1, 2)  // -1  (1 < 2)
OrdNumber.compare(2, 2)  //  0  (equal)
OrdNumber.compare(3, 2)  //  1  (3 > 2)
```

### Ord.contramap — the most-used Ord combinator

Derive `Ord<B>` from `Ord<A>` by providing a function `B => A`:

```typescript
import { pipe } from 'fp-ts/function'
import * as A from 'fp-ts/Array'

type Employee = { name: string; department: string; salary: number }

// Sort employees by department (alphabetically)
const OrdByDept: Ord.Ord<Employee> = pipe(
  OrdString,
  Ord.contramap((e: Employee) => e.department)
)

// Sort employees by salary (ascending)
const OrdBySalary: Ord.Ord<Employee> = pipe(
  OrdNumber,
  Ord.contramap((e: Employee) => e.salary)
)

const employees: Employee[] = [
  { name: 'Alice', department: 'Engineering', salary: 95000 },
  { name: 'Bob',   department: 'Design',      salary: 85000 },
  { name: 'Carol', department: 'Engineering', salary: 105000 },
]

const byDept   = A.sort(OrdByDept)(employees)
// [{ Bob, Design }, { Alice, Engineering }, { Carol, Engineering }]

const bySalary = A.sort(OrdBySalary)(employees)
// [{ Bob, 85000 }, { Alice, 95000 }, { Carol, 105000 }]
```

### Ord.reverse — flip the ordering

```typescript
const OrdBySalaryDesc: Ord.Ord<Employee> = Ord.reverse(OrdBySalary)
const highestFirst = A.sort(OrdBySalaryDesc)(employees)
// [{ Carol, 105000 }, { Alice, 95000 }, { Bob, 85000 }]
```

### Composing multiple Ord — sort by lastName then firstName

```typescript
import { getMonoid } from 'fp-ts/Ord'
import * as M from 'fp-ts/Monoid'

type Person = { firstName: string; lastName: string }

const OrdByLastName: Ord.Ord<Person> = pipe(
  OrdString,
  Ord.contramap((p: Person) => p.lastName)
)

const OrdByFirstName: Ord.Ord<Person> = pipe(
  OrdString,
  Ord.contramap((p: Person) => p.firstName)
)

// Combine: sort by lastName first, then firstName for ties
// Ord has a Semigroup: if first comparison is 0 (equal), use the second
const OrdByFullName: Ord.Ord<Person> = M.concatAll(getMonoid<Person>())([
  OrdByLastName,
  OrdByFirstName,
])

const people: Person[] = [
  { firstName: 'Alice', lastName: 'Smith' },
  { firstName: 'Bob',   lastName: 'Jones' },
  { firstName: 'Carol', lastName: 'Smith' },
]

A.sort(OrdByFullName)(people)
// [{ Bob, Jones }, { Alice, Smith }, { Carol, Smith }]
```

## Semigroup — Combining Two Values

A `Semigroup<A>` has one method: `concat(a: A, b: A): A`. It defines how to combine two values of the same type. The operation must be associative: `concat(concat(a, b), c) === concat(a, concat(b, c))`.

```typescript
import * as S from 'fp-ts/Semigroup'
import { Semigroup as SemigroupString } from 'fp-ts/string'
import { Semigroup as SemigroupNumber } from 'fp-ts/number' // not exported — use Sum/Product

// Built-in string Semigroup: concat = string concatenation
SemigroupString.concat('Hello, ', 'World') // 'Hello, World'

// Sum Semigroup for numbers
import * as N from 'fp-ts/number'
// Use S.concatAll with a Semigroup and a list
```

### Common Semigroup instances

```typescript
import { first, last } from 'fp-ts/Semigroup'

// first: always keep the left value
const firstNum = first<number>()
firstNum.concat(1, 2) // 1

// last: always keep the right value
const lastNum = last<number>()
lastNum.concat(1, 2) // 2
```

### S.concatAll — fold an array with a Semigroup

```typescript
import { SemigroupSum } from 'fp-ts/number'

// Sum all numbers
const total = S.concatAll(SemigroupSum)(0)([1, 2, 3, 4, 5])
// Note: SemigroupSum may not be exported — use Monoid.concatAll for this pattern
```

## Monoid — Semigroup + Identity Element

A `Monoid<A>` is a `Semigroup<A>` with an `empty: A` value (the identity element), such that:
- `concat(a, empty) === a` for all `a`
- `concat(empty, a) === a` for all `a`

```typescript
import * as Mo from 'fp-ts/Monoid'
import { MonoidSum, MonoidProduct } from 'fp-ts/number'
import { Monoid as MonoidString } from 'fp-ts/string'

MonoidString.empty              // ''
MonoidString.concat('a', 'b')  // 'ab'
MonoidString.concat('x', '')   // 'x' (identity holds)

MonoidSum.empty              // 0
MonoidSum.concat(3, 4)       // 7
MonoidSum.concat(5, 0)       // 5 (identity holds)

MonoidProduct.empty          // 1
MonoidProduct.concat(3, 4)   // 12
MonoidProduct.concat(5, 1)   // 5 (identity holds)
```

### Mo.concatAll — the primary Monoid use case

```typescript
Mo.concatAll(MonoidSum)([1, 2, 3, 4, 5])      // 15
Mo.concatAll(MonoidString)(['a', 'b', 'c'])    // 'abc'
Mo.concatAll(MonoidProduct)([2, 3, 4])         // 24
Mo.concatAll(MonoidSum)([])                    // 0 (uses empty for the base case)
```

The key advantage over `Array.reduce`: `Mo.concatAll` handles the empty array case via `empty`, so you never need a fallback.

## When to Use / When NOT to Use

**Use these typeclasses when:**

- You need to sort arrays of custom objects — `Ord.contramap` + `A.sort` is the canonical pattern
- You need structural equality for custom types — `Eq.fromEquals` or `Eq.contramap`
- You want to aggregate a collection of values in a generic way — `Mo.concatAll`
- You are building utility functions that need to work across multiple types (generic programming)
- You are reading fp-ts library code — understanding Ord/Eq unlocks Option.getEq, Array.sort, Map, etc.

**Do NOT use these when:**

- **Built-in operations suffice.** For a simple array of numbers, `arr.sort((a, b) => a - b)` is perfectly readable. Pull in Ord when you need composability or type-class generic code.
- **The typeclass overhead exceeds the benefit.** Building a custom `Monoid` for a one-off reduce is over-engineering. Use `Array.reduce` directly.
- **You need partial order or non-associative combination.** These typeclasses are for total orders and associative operations. Non-associative combining requires a different abstraction.

## Real-World Example

Building a leaderboard sort for a game with multiple tiebreakers:

```typescript
import { pipe } from 'fp-ts/function'
import * as A from 'fp-ts/Array'
import * as Ord from 'fp-ts/Ord'
import * as Mo from 'fp-ts/Monoid'
import { Ord as OrdNumber } from 'fp-ts/number'
import { Ord as OrdString } from 'fp-ts/string'
import { getMonoid } from 'fp-ts/Ord'

type Player = {
  name:       string
  score:      number
  wins:       number
  joinedDate: string  // ISO string, lexicographically sortable
}

// Build Ord for each tiebreaker — most important first
const OrdByScore: Ord.Ord<Player> = pipe(
  OrdNumber,
  Ord.contramap((p: Player) => p.score),
  Ord.reverse   // higher score = better rank
)

const OrdByWins: Ord.Ord<Player> = pipe(
  OrdNumber,
  Ord.contramap((p: Player) => p.wins),
  Ord.reverse   // more wins = better rank
)

const OrdByJoinDate: Ord.Ord<Player> = pipe(
  OrdString,
  Ord.contramap((p: Player) => p.joinedDate)
  // earlier date = longer member = better rank — no reverse needed
)

// Combine: score first, wins for ties, join date for remaining ties
const OrdLeaderboard: Ord.Ord<Player> = Mo.concatAll(getMonoid<Player>())([
  OrdByScore,
  OrdByWins,
  OrdByJoinDate,
])

const players: Player[] = [
  { name: 'Alice', score: 1500, wins: 10, joinedDate: '2023-01-15' },
  { name: 'Bob',   score: 1500, wins: 12, joinedDate: '2023-06-01' },
  { name: 'Carol', score: 2000, wins: 20, joinedDate: '2024-01-01' },
  { name: 'Dave',  score: 1500, wins: 10, joinedDate: '2022-09-01' },
]

const leaderboard = A.sort(OrdLeaderboard)(players)
// 1. Carol (2000 score)
// 2. Bob   (1500 score, 12 wins)
// 3. Dave  (1500 score, 10 wins, joined 2022 — earlier)
// 4. Alice (1500 score, 10 wins, joined 2023 — later)
```

Notice how each tiebreaker is a composable `Ord` instance. Adding a new tiebreaker is one line — no rewriting of comparison logic.

## Key Takeaways

- `Eq<A>` defines structural equality — use `Eq.fromEquals` to create custom instances, `Eq.contramap` to derive from existing ones
- `Ord<A>` extends `Eq<A>` with ordering — use `Ord.contramap` to sort by any field, `Ord.reverse` to invert, and `Mo.concatAll(getMonoid<A>())` to compose multiple tiebreakers
- `Semigroup<A>` defines associative combining (`concat`) — `first`/`last` are the most useful built-in instances
- `Monoid<A>` is a `Semigroup<A>` plus an identity element (`empty`) — `Mo.concatAll` handles empty arrays automatically
- These typeclasses are the foundation that makes fp-ts's generic operations work — understanding them explains why `A.sort`, `O.getEq`, and `M.union` accept type class instances
